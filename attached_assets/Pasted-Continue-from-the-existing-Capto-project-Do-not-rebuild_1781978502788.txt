Continue from the existing Capto project. Do not rebuild authentication, task management, database structure, admin pages, mobile task browsing, OTP flow, or existing navigation.

Build the mobile in-app capture and local draft workflow only.

Important scope boundary:

* Do not upload media to backend or cloud storage yet.
* Do not create final submissions yet.
* Do not change wallet logic.
* Do not build admin review changes.
* This module should record or capture media locally, validate it against task requirements, save it as a local draft, and allow the user to review/delete/retry it.

Technology:

* Expo React Native with TypeScript
* Expo Router
* expo-camera
* expo-file-system for local media metadata and draft management
* expo-audio for audio recording if needed
* Use existing state-management pattern already in the project

3. Video capture screen
   Use expo-camera CameraView and create a polished full-screen in-app recording experience.

Requirements:

* Rear camera by default unless task preferredCamera is FRONT.
* Allow front/rear switch only when preferredCamera is ANY.
* Respect preferredLens in UI as much as Expo supports:

  * If preferredLens is ULTRA_WIDE, set lowest practical zoom and display “Wide lens requested”.
  * Do not claim guaranteed physical 0.5x ultra-wide support.
  * Show fallback message if the device cannot provide wide-angle behavior.
* Use task preferredFps / minimumFps as requirement labels only for now.
* Do not falsely report exact FPS if Expo does not expose reliable device FPS metadata.
* Use device-supported camera settings and gracefully fall back.

Controls:

* Live camera preview
* Recording timer
* Recording red indicator
* Start recording button
* Stop recording button
* Pause/resume buttons when supported
* Flash/torch toggle
* Optional camera switch button based on task rules
* Close/cancel button with confirmation if recording has started

Validation:

* On stop, validate duration against:

  * minimumDurationSeconds
  * maximumDurationSeconds
* Show a clear error if too short or too long.
* Allow user to retake immediately.

Draft fields:

* localDraftId

* taskTitle
* categoryId
* subcategoryId
* collectionType
* localMediaUri or mediaUris
* thumbnailUri where available
* durationSeconds where applicable
* fileSize where available
* createdAt
* device platform

* orientation
* capture metadata object
* validation status

Store draft metadata locally using the existing preferred local-storage approach. Use file-system-safe local paths for media.
