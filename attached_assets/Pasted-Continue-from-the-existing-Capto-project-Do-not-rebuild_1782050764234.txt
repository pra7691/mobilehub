Continue from the existing Capto project. Do not rebuild capture, upload, task management, wallet, payout, referral, language, or existing admin layout.

The Admin Dashboard currently shows submitted tasks/submissions, but there is no action to approve, reject, or change submission status.

Build the complete Admin Submission Review workflow.

## Goal

Admin must be able to review every submitted task and:

* Approve submission
* Reject submission
* Request resubmission

When approved:

* Submission status changes to APPROVED.
* User receives the task payment amount in wallet.
* Wallet transaction is created.
* If the user was referred and this is their first approved task, trigger referral reward logic if that module exists.
* Mobile user sees the submission in Completed and wallet balance updates.

When rejected:

* Submission status changes to REJECTED.
* Admin must enter rejection reason.
* No task payment is credited.
* Mobile user sees rejection reason.

When resubmission is requested:

* Submission status changes to RESUBMISSION_REQUIRED.
* Admin must enter reason.
* No task payment is credited.
* Mobile user sees action-needed status and reason.

## 3. Backend APIs

Ensure these admin-only APIs exist and are fully wired:

* GET /admin/submissions
* GET /admin/submissions/:id
* POST /admin/submissions/:id/approve
* POST /admin/submissions/:id/reject
* POST /admin/submissions/:id/request-resubmission

Approval request body:

```json
{
  "approvedAmount": 40,
  "adminNote": "Approved after quality review."
}
```

Reject request body:

```json
{
  "rejectionReason": "Video is too dark and instructions are not visible.",
  "adminNote": "Please record again in better lighting."
}
```

Resubmission request body:

```json
{
  "resubmissionReason": "Please capture the full activity from start to finish.",
  "adminNote": "Do not skip the preparation steps."
}
```

Rules:

* Only submissions with status UNDER_REVIEW can be reviewed.
* Return clear error if already reviewed.
* Prevent duplicate approval.
* Do not allow double wallet credit.
* Use database transaction for approval and wallet update.
* Save reviewedBy admin ID and reviewedAt.
* Save approved amount, rejection reason, resubmission reason, and admin note where applicable.
* Add Swagger docs and DTO validation.

## 4. Wallet Integration

On approval:

1. Lock submission record.
2. Confirm submission is UNDER_REVIEW.
3. Change submission status to APPROVED.
4. Set approvedAmount.
5. Create WalletTransaction:

   * transactionType: CREDIT
   * sourceType: SUBMISSION
   * sourceId: submission ID
   * amount: approvedAmount
   * note: task title / submission ID
6. Increase user available balance.
7. Increase user lifetime earnings.
8. Save wallet balance before and after.
9. Ensure this can happen only once.

If referral system exists:

* Trigger referral reward only after normal submission credit succeeds.
* Reward referrer only once when referred user gets their first approved task.
* Do not block normal approval if referral reward fails; log it safely and make it recoverable.

On reject or resubmission:

* Do not create wallet credit.
* Do not change available balance.
